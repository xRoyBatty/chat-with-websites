## Core Strategy

### The Fundamental Insight

**Don't type more. Type smarter.**

Your bottleneck isn't the AI agent - it's YOU. With unlimited agent access, the constraint is your:
- Typing speed
- Decision-making speed
- Context-switching time
- Sleep requirements

**Solution:** Maximize the agent's autonomous work, minimize your active time.

---

### The 3-Phase Approach

**Phase 1: Setup (Day 1)**
- Create infrastructure for autonomous operation
- Build delegation systems
- Establish monitoring dashboards

**Phase 2: Execute (Days 2-6)**
- Agent works 24/7
- You work 2-4 hours/day (high-leverage activities)
- Everything else runs autonomously

**Phase 3: Harvest (Day 7)**
- Consolidate results
- Deploy products
- Archive knowledge
- Set up sustainability

## Background Process Orchestration

### Master Orchestrator Pattern

**The problem:** Too many parallel tasks to track manually

**The solution:** Self-managing task orchestration

**Concept:**
- Orchestrator runs continuously in background
- Reads task queue periodically
- Starts pending tasks automatically
- Monitors running tasks
- Detects completion
- Sends notifications
- Starts dependent tasks when prerequisites complete
- Generates status dashboards
- All happens without human intervention

**Your interaction:**
- Queue a task in seconds by adding to queue file
- Check status anytime by viewing status files
- View comprehensive dashboard in browser

**Result:**
- Queue 50+ tasks initially
- All execute automatically over the week
- You just monitor dashboard
- Get notified when important things complete

**Key benefit:** Set up once, runs continuously, eliminates need for constant task management.

---

### Dependency Chain Automation

**Complex workflows with dependencies:**

**Concept:**
Define workflows where tasks have prerequisites and dependencies. The orchestrator automatically:
1. Starts tasks with no dependencies
2. Waits for completion
3. Starts dependent tasks automatically
4. Handles parallel execution when possible
5. Manages approval gates for critical steps
6. Notifies you only when human decision needed

**Example flow:**
- Design database schema (starts immediately)
- Generate backend API (waits for schema)
- Generate frontend (waits for API)
- Write tests (waits for both backend and frontend)
- Deploy (waits for tests, requires your approval)

**Your involvement:**
- Day 1: Create workflow definition (15 min)
- Day 3: Approve deployment (5 min)
- Total: 20 minutes for entire full-stack app

**Coordination benefit:** Complex multi-day projects execute automatically with dependency management.

**Don't create once. Create templates that generate infinitely.**

**Concept:**

Instead of creating one instance of something, create a system that generates many:

Example: Instead of writing 1 blog post, create a system that generates 100:
- Day 1: Create blog post generator skill with research, writing, SEO optimization

**Key insight:** Build systems, not individual outputs.

### Tactic 2: The Snowball Strategy

**Start small, compound daily.**

**Concept:**

Day 1: Learn topic A (active for 1 hour)
- Agent creates extensive materials

Day 2: Review topic A (30 min), Learn topic B (30 min)
- Agent creates materials for B
- Agent creates advanced materials for A (now that you know basics)

Day 3: Review A+B (30 min), Learn topic C (30 min)
- Agent creates materials for C
- Agent creates advanced materials for A and B
- Agent finds connections between A, B, C

... by Day 7:
- You've learned 7 topics (minimal active hours)
- Agent created extensive materials for all
- Agent created advanced materials
- Agent identified cross-topic connections
- You have expert-level understanding of all 7

**Comparison:**
- Normal learning: 7 topics requiring extensive time each
- With agent: 7 topics in minimal time through automated material creation
- Massive speedup through automation
