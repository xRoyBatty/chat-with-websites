# User Profiles

## Purpose
Profiles help match users to appropriate solutions based on their use case, needs, and context.

## How to Use Profiles
1. During consultation, identify which profile(s) match the user
2. Read the corresponding profile file for:
   - Typical needs and pain points
   - Recommended templates
   - Tool suggestions
   - Implementation priorities
3. Customize based on specific user answers

## Available Profiles

### Research & Learning
- **researcher-student.md** - Academic researchers, graduate students, professional researchers
- **language-learner.md** - Language students, polyglots, self-taught learners

### Professional & Productivity
- **busy-professional.md** - Working professionals juggling multiple goals and projects
- **content-creator.md** - Writers, educators, documentation specialists

### Development
- **hobbyist-developer.md** - Amateur developers, weekend hackers, learning to code

## Profile Structure
Each profile includes:
- **Who they are** - Background and context
- **Common pain points** - What frustrates them
- **Typical needs** - What they're trying to accomplish
- **Recommended templates** - Best starting points
- **Tool suggestions** - Which advanced tools would help
- **Success metrics** - How to measure value
- **Warning signs** - When this profile doesn't fit

## Using Profiles in Consultation

**Step 1:** Identify potential profile match
- Listen for keywords and pain points
- May match multiple profiles
- Fine to combine profiles

**Step 2:** Validate the match
- Ask clarifying questions
- Confirm assumptions
- Adjust for unique needs

**Step 3:** Apply profile guidance
- Recommend appropriate templates
- Suggest relevant tools
- Set expectations correctly

## Example Profile Combinations

**Graduate Student + Hobbyist Developer:**
- Needs: Research management + Building analysis tools
- Solution: Knowledge base + VPS for running analysis scripts
- Templates: personal-knowledge-base + custom dev setup

**Busy Professional + Content Creator:**
- Needs: Goal tracking + Documentation generation
- Solution: Integrated Life OS with goals + content domains
- Templates: life-os with focus on productivity + content

**Language Learner + Researcher:**
- Needs: Vocabulary tracking + Cultural research
- Solution: Knowledge base with language learning extensions
- Templates: personal-knowledge-base customized for language

## Notes

- Profiles are guidelines, not rigid categories
- Most users are unique combinations
- Use profiles for inspiration, not prescription
- Always customize based on actual user needs
